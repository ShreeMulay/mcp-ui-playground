TL;DR - Executive Summary

What is MCP-UI? SDK for building interactive UI in AI chat apps (not an MCP server itself)
Server SDK Creates UIResource objects (HTML/URLs) - available in TypeScript, Python, Ruby
Client SDK Renders UIResources in React/Web apps via sandboxed iframes
OpenCode? Can BUILD MCP-UI code, cannot RENDER it (CLI vs browser)
Best GCP Option Cloud Run (scales to zero, ~$0 at low traffic)
HIPAA? MCP-UI doesn't handle it - you implement at infrastructure layer

The UIResource Object

Think of it as a shipping box with a label:

{
  type: 'resource',
  resource: {
    uri: 'ui://my-form/123',        // Address label
    mimeType: 'text/html',          // What's inside
    text: '<button>Click</button>' // The actual content
  }
}
mimeType Content Type Use Case
text/html Direct HTML Forms, buttons, charts
text/uri-list External URL Embed dashboards, apps
application/vnd.mcp-ui.remote-dom Remote DOM JS Complex interactive apps

What OpenCode Can and Can't Do

Task Can Do? Notes
Write MCP-UI server code Yes Full code generation
Write MCP-UI client code (React) Yes Components, handlers, etc.

OpenCode vs Claude.ai: Same AI model, different wrapper. OpenCode excels at real projects (file access, terminal commands). Claude.ai Artifacts are better for quick visual prototypes.

HIPAA Compliance

MCP-UI Does NOT Handle HIPAA. HIPAA compliance depends on your entire stack - authentication, encryption, logging, and vendor agreements.

HIPAA Requirements Overview

Requirement What It Means Where to Implement
PHI Protection Protect patient health info at rest and in transit Database, API, Network
Access Controls Only authorized users see data Auth layer (Firebase, etc.)
Audit Logs Track who accessed what, when Cloud Audit Logs
BAA Business Associate Agreement with vendors Sign with GCP
Encryption Data encrypted everywhere Cloud SQL, Storage, TLS
